## How to Contribute

1. **Fork the repository** to your personal GitHub account.
2. **Clone your fork locally**:
   ```bash
   git clone git@github.com:<your-username>/altimetry-processing-pipeline.git
3. **Create a new branch for your feature or fix**:
    ```bash
    git checkout -b feature/my-new-feature
4. **Make your changes, then commit following clear messages**:
    ```bash
    git add <files>
    git commit -m "Brief description of changes"
5. **Push your branch to your fork**:
    ```bash 
    git push origin feature/my-new-feature
6. **Open a Pull Request (PR) against main of this repo.**

## Reporting Issues
- Use the Issues tab to report bugs or request features.
- Provide clear steps to reproduce, expected vs. actual behavior, and relevant logs or screenshots.

## Pull Request Workflow
1. PRs should be targeted to main.
2. Include a clear description of the changes and motivation.
3. Ensure all tests pass (if applicable) and adhere to the coding guidelines below.
4. PRs will be reviewed by maintainers before merging.

## Coding Guidelines
1. Follow Python PEP 8 style.
2. Include docstrings for all public functions and classes.
